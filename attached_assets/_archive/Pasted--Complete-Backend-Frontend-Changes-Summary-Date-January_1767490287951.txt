### 1.2 Drizzle WHERE Clause Bug

**File:** `backend/storage/clients.ts`

**Method:** `getPaginatedClients()`

**Issue:** `.where()` calls were REPLACING previous filters instead of combining with AND.

**Lines Changed:** ~85-97

**BEFORE:**
```typescript
let query = db.select().from(clients).where(eq(clients.companyId, companyId));
if (options.inactive !== undefined) {
  query.where(eq(clients.inactive, false)); // ❌ REPLACES companyId filter
}
```

**AFTER:**
```typescript
const whereConditions = [eq(clients.companyId, companyId)];
if (options.inactive !== undefined) {
  whereConditions.push(eq(clients.inactive, options.inactive));
} else {
  whereConditions.push(eq(clients.inactive, false));
}
let query = db.select().from(clients).where(and(...whereConditions));
```

**Impact:** Prevents cross-tenant data leakage in paginated client lists.

## 2. Soft Delete Implementation

**File:** `backend/storage/clients.ts`

**Methods Modified:**
- `deleteClient()` (lines ~242-249)
- `deleteClients()` (lines ~254-271)

**Change:** Hard delete → Soft delete (set `inactive = true`)

**BEFORE:**
```typescript
async deleteClient(companyId: string, clientId: string): Promise<boolean> {
  const result = await db
    .delete(clients)
    .where(and(eq(clients.id, clientId), eq(clients.companyId, companyId)))
    .returning();
  return result.length > 0;
}
```

**AFTER:**
```typescript
async deleteClient(companyId: string, clientId: string): Promise<boolean> {
  this.assertCompanyId(companyId);
  this.validateUUID(clientId, "clientId");

  const rows = await db
    .update(clients)
    .set({ 
      inactive: true,
      updatedAt: new Date() 
    })
    .where(and(eq(clients.id, clientId), eq(clients.companyId, companyId)))
    .returning();

  return rows.length > 0;
}
```

**Impact:** 
- Preserves referential integrity (jobs/invoices remain linked)
- Enables data recovery
- Maintains audit trail

## 3. Deterministic Ordering

**File:** `backend/storage/clients.ts`

**Methods Modified:**
- `getAllClients()` (lines ~33-39)
- `getPaginatedClients()` (lines ~89-97)

**Change:** Single-column sort → Multi-column deterministic sort

**getAllClients() AFTER:**
```typescript
async getAllClients(companyId: string): Promise<Client[]> {
  return await db
    .select()
    .from(clients)
    .where(eq(clients.companyId, companyId))
    .orderBy(
      clients.companyName,              // Primary

    );
}
```

**getPaginatedClients() AFTER:**

**Database Index Added:**


**Impact:** Consistent UI ordering across page refreshes.

## 6. Signup Route Implementation

**File:** `backend/routes/auth.ts`

**Route Added:** `POST /api/auth/signup` (after line ~90)

**Features:**
- User registration with email/password
- Automatic company creation for new signups
- Invitation token support for joining existing companies
- Auto-login after successful signup
- Role assignment (admin for new companies, technician for invitations)

**Full Implementation:**
```typescript
router.post("/signup", async (req: Request, res: Response) => {
  try {
    const { email, password, firstName, lastName, invitationToken } = req.body;

    if (!email || !password) {
      return res.status(400).json({ error: "Email and password are required" });
    }

    const existingUser = await storage.getUserByEmail(email);
    if (existingUser) {
      return res.status(400).json({ error: "User already exists" });
    }

    let companyId: string;
    let role = "admin";

    if (invitationToken) {
      const invitation = await storage.getInvitationByToken(invitationToken);
      if (!invitation || invitation.status !== 'pending') {
        return res.status(400).json({ error: "Invalid or expired invitation" });
      }
      companyId = invitation.companyId;
      role = invitation.role || "technician";
      await storage.updateInvitation(invitation.id, { status: 'accepted' });
    } else {
      const newCompany = await storage.createCompany({
        name: email.split('@')[0] + "'s Company",
        email: email,
      });
      companyId = newCompany.id;
    }

    const user = await storage.createUser({
      email,
      password,
      companyId,
      role,
      firstName,
      lastName,
    });

    req.logIn(user, (err) => {
      if (err) {
        return res.status(500).json({ error: "Account created but login failed" });
      }
      res.status(201).json({
        id: user.id,
        email: user.email,
        role: user.role,
        companyId: user.companyId,
      });
    });
  } catch (error: any) {
    console.error("[AUTH] Signup error:", error);
    res.status(500).json({ error: error.message || "Failed to create account" });
  }
});
```

**Dependencies Added:**
```typescript
import { storage } from "../storage/index";
```

**Impact:** User registration flow now works end-to-end.

## 8. Customer Companies Location Creation

**File:** `backend/routes/customer-companies.ts`

**Route Added:** `POST /:companyId/locations` (after line ~69, before GET overview route)

**Purpose:** Create new locations under customer companies

**Full Implementation:**
```typescript
router.post("/:companyId/locations", async (req: AuthedRequest, res: Response) => {
  try {
    const { companyId: tenantCompanyId, user } = req;
    const { companyId } = req.params;

    if (!tenantCompanyId || !user) {
      return res.status(401).json({ error: "Unauthorized" });
    }

    const [company] = await db
      .select()
      .from(customerCompanies)
      .where(and(eq(customerCompanies.id, companyId), eq(customerCompanies.companyId, tenantCompanyId)))
      .limit(1);

    if (!company) {
      return res.status(404).json({ error: "Customer company not found" });
    }

    const [newLocation] = await db
      .insert(clients)
      .values({
        companyId: tenantCompanyId,
        userId: user.id,
        parentCompanyId: companyId,
        companyName: company.name,
        location: req.body.location || "",
        address: req.body.address || null,
        city: req.body.city || null,
        province: req.body.province || null,
        postalCode: req.body.postalCode || null,
        contactName: req.body.contactName || null,
        email: req.body.email || null,
        phone: req.body.phone || null,
        roofLadderCode: req.body.roofLadderCode || null,
        billWithParent: req.body.billWithParent ?? true,
        inactive: req.body.inactive ?? false,
        isPrimary: false,
        needsDetails: false,
      })
      .returning();

    res.status(201).json(newLocation);
  } catch (err: any) {
    console.error("Create location error:", err);
    res.status(500).json({ error: err?.message || "Failed to create location" });
  }
});
```

**Impact:** Locations can now be created under customer companies via UI.

## Critical Patterns to Follow Going Forward

### 1. Always Use `and()` for Multiple WHERE Conditions
```typescript
// ✅ CORRECT
const conditions = [eq(table.companyId, companyId)];
if (filter) conditions.push(eq(table.field, value));
query.where(and(...conditions));

// ❌ WRONG - Replaces previous WHERE
query.where(eq(table.companyId, companyId));
query.where(eq(table.field, value));
```

### 2. Always Soft Delete Business Data
```typescript
// ✅ CORRECT
await db.update(table).set({ inactive: true }).where(...);

// ❌ WRONG - Loses data
await db.delete(table).where(...);
```

### 3. Always Add Deterministic Ordering
```typescript
// ✅ CORRECT
.orderBy(table.name, table.createdAt, table.id)

// ❌ WRONG
.orderBy(table.name)
```

### 4. Always Use Optional Chaining for Potentially Undefined Data
```typescript
// ✅ CORRECT
companyId={location?.companyId || ""}

// ❌ WRONG
companyId={location.companyId}
```

### 5. Scope Middleware Correctly
```typescript
// ✅ CORRECT
app.use("/api", requireAuth);

// ❌ WRONG
app.use(requireAuth);
```
