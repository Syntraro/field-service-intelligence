3. Status Transitions & Close Logic
Problem Fixed

Invalid transitions like:

scheduled → archived
scheduled → invoiced

Solution

New atomic endpoint:

POST /api/jobs/:id/close
{ mode: archive | invoice_later | invoice_now }

Behavior

Internally performs:

current → completed → final_state


Validated by assertJobStatusTransition.

For invoice_now, invoice creation + job update occur atomically.

4. Reopen + Undo Close
Reopen

Allowed from:

completed

archived

invoiced (must void/credit invoice)

Undo Close (20s window)

Fields:

previousStatus

closedBy

Endpoint:

POST /api/jobs/:id/undo-close


Rules:

≤ 20 seconds

Not invoiced

Restores previous status

Clears undo fields

UI shows Undo toast for archive / invoice_later.
