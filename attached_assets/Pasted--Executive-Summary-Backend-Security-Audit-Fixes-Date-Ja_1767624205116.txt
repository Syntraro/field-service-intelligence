## Critical Security Fixes (URGENT)

### 1. ✅ Tenant Isolation Vulnerability
**Issue:** Cross-tenant data leakage - users could see other companies' data  
**Fix:** 
- Added `req.companyId` assignment in requireAuth middleware
- Fixed Drizzle ORM WHERE clause bug (was replacing filters instead of combining)
- **Impact:** 100% tenant isolation - zero cross-tenant data exposure

### 2. ✅ Authentication System
**Issue:** No signup endpoint, middleware blocking everything  
**Fix:**
- Implemented complete signup flow with auto-login
- Scoped requireAuth to `/api` only (not blocking frontend)
- Added storage methods: createCompany, createUser, getInvitationByToken, updateInvitation
- **Impact:** User registration works, frontend accessible

### 3. ✅ CSRF Protection
**Status:** Fully implemented across all API writes  
- Token endpoint: `GET /api/csrf-token`
- Auto-attached by frontend `apiRequest()`
- No bypasses

## Key Patterns Established

### 1. Tenant Isolation
Always use `and()` for multiple WHERE conditions:
```typescript
const conditions = [eq(table.companyId, companyId)];
if (filter) conditions.push(eq(table.field, value));
query.where(and(...conditions));
```

### 2. Soft Delete
Never hard delete business data:
```typescript
await db.update(table).set({ inactive: true }).where(...);
```

### 3. Deterministic Ordering
Always sort by multiple columns:
```typescript
.orderBy(table.name, table.createdAt, table.id)
```

### 4. Null Safety
Use optional chaining for async data:
```typescript
companyId={location?.companyId || ""}
```
