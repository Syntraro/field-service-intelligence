### 6. TRANSACTION BOUNDARIES (CRITICAL - COMPLETE)

### 8. LOGIN RATE LIMITING (CRITICAL - COMPLETE)
**Problem:** No brute force protection  
**Solution:** Two-layer rate limiting with account lockout  

**Protection:**
- Layer 1: 5 attempts per 15 minutes (IP-based)
- Layer 2: 10 attempts per hour (email-based account lockout)
- Generic error messages (no user enumeration)
- Security logging (all attempts logged)

**Files:**
- `server/routes/auth.ts` - Rate limiting middleware
- Dependencies: `express-rate-limit` (npm package installed)

### 10. OPTIMISTIC LOCKING (CRITICAL - COMPLETE)
**Problem:** Two users editing same record causes lost updates  
**Solution:** Version-based optimistic locking  

**Implementation:**
- Added `version INTEGER` column to invoices, jobs, clients
- Storage layer checks version before update, increments on success
- Routes extract version from request, return 409 on conflict
- Backward compatible (version parameter optional)

**Impact:** Prevents lost updates in concurrent edits, user-friendly errors

**Files:**
- `migrations/005-add-optimistic-locking.sql`
- `shared/schema.ts` - Added version columns
- `server/storage/invoices.ts` - Version checking
- `server/storage/jobs.ts` - Version checking
- `server/storage/clients.ts` - Version checking
- `server/routes/invoices.ts` - Handle 409 conflicts
- `server/routes/jobs.ts` - Handle 409 conflicts
- `server/routes/clients.ts` - Handle 409 conflicts

### 11. MONEY TYPE FIX (CRITICAL - COMPLETE)
**Problem:** Financial data stored as TEXT causing calculation errors  
**Solution:** Migrated to NUMERIC(12,2) for proper decimal precision  

**Tables Fixed (13 tables):**
- Companies: `defaultTaxRate` TEXT → NUMERIC(5,2)
- Parts: `cost`, `unitPrice`, `markupPercent` TEXT → NUMERIC(12,2)
- Invoices: `subtotal`, `taxTotal`, `total`, `amountPaid`, `balance` TEXT → NUMERIC(12,2)
- Invoice Lines: `unitCost`, `unitPrice`, `lineSubtotal` TEXT → NUMERIC(12,2), `taxRate` TEXT → NUMERIC(5,4)
- Payments: `amount` TEXT → NUMERIC(12,2)
- Job Parts: `unitCost`, `unitPrice` TEXT → NUMERIC(12,2)
- Technician Profiles: `laborCostPerHour`, `billableRatePerHour` TEXT → NUMERIC(8,2)
- Job Template Line Items: `unitPriceOverride` TEXT → NUMERIC(12,2)

**Impact:** 
- Proper decimal precision
- No string concatenation bugs
- Database can do SUM/AVG correctly
- No JavaScript floating point errors
- Simpler code (no parseFloat everywhere)

**Files:**
- `migrations/006-fix-money-types-FIXED.sql`
- `shared/schema.ts` - Updated to use `numeric()` types
- `server/storage/invoices.ts` - Removed parseFloat/toFixed
- `MONEY-TYPE-MIGRATION-GUIDE.md` - Complete guide

## 🎓 KEY TECHNICAL DECISIONS

### 1. Chose Optimistic Locking over Pessimistic
**Why:** Better scalability, no lock contention, works well for low-conflict scenarios

### 2. Used In-Memory Cache (not Redis)
**Why:** Sufficient for single-server deployment, simpler to maintain
**When to Migrate:** Only if running multiple server instances

### 3. Kept Pagination Optional
**Why:** Backward compatibility, gradual rollout
**Default:** 50 items per page, max 200

### 4. Made Version Checks Optional
**Why:** Backward compatibility, allows gradual frontend updates
**Recommendation:** Frontend should track versions for critical operations
